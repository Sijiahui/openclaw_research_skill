# Step 6: 验证思考文档质量

## 目标

强制验证每日思考文档的质量，确保包含所有必需内容。

## 核心原则

```
╔════════════════════════════════════════════════════════════╗
║  🔍 强制验证：生成后必须执行验证脚本                       ║
║                                                              ║
║  ✅ 验证通过 → 继续Git提交                                  ║
║  ❌ 验证失败 → 重新生成，补充缺失部分                      ║
║                                                              ║
║  🚨 如果验证失败3次，强制停止任务并报告错误               ║
╚════════════════════════════════════════════════════════════╝
```

## 执行命令

```bash
# 验证今天的思考文档
bash ~/.openclaw/workspace/skills/spatial-agi-research/scripts/validate_thinking.sh

# 如果验证失败，查看详细错误
bash ~/.openclaw/workspace/skills/spatial-agi-research/scripts/validate_thinking.sh $(date +%Y-%m-%d)
```

## 验证内容（11项检查）

### 检查1: 文档存在性
- ✅ 文件存在
- ❌ 文件不存在

### 检查2: 文档行数
- ✅ ≥ 800行
- ❌ < 800行

### 检查3: 与昨日思考的联系
- ✅ 包含章节"🔗 与昨日思考的联系"
- ❌ 缺少该章节

### 检查4: 核心见解演进图
- ✅ 包含Mermaid `graph LR`代码
- ❌ 缺少演进图

### 检查5: 技术栈演进对比表
- ✅ 包含表格（至少5行）
- ❌ 缺少表格

### 检查6: 知识缺口分析
- ✅ 包含"知识缺口分析"章节
- ✅ 包含Mermaid `pie`图
- ❌ 缺少pie图

### 检查7: 10层架构思维导图
- ✅ 包含"10层架构思维导图"
- ✅ 包含Mermaid `mindmap`代码
- ✅ 至少10层（Level 0-9）
- ❌ 缺少思维导图

### 检查8: 主线技术路径
- ✅ 包含"主线技术路径"
- ✅ 至少4个阶段
- ❌ 阶段不足

### 检查9: 待探索议题
- ✅ 包含"待探索议题"
- ✅ 至少9个议题
- ❌ 议题数量不足

### 检查10: 本质思考
- ✅ 包含"💡 本质思考"
- ✅ 包含3个子问题
- ❌ 缺少子问题

### 检查11: Mermaid图表总数
- ✅ ≥ 3个图表
- ❌ < 3个图表

## 失败处理流程

### 第1次失败

```bash
echo "⚠️  验证失败，需要补充内容"
# 查看具体哪些检查失败
# 根据错误信息补充缺失部分
# 再次验证
bash ~/.openclaw/workspace/skills/spatial-agi-research/scripts/validate_thinking.sh
```

### 第2次失败

```bash
echo "⚠️  第2次验证失败，重新检查"
# 仔细检查错误信息
# 手动补充缺失内容
# 第3次验证
bash ~/.openclaw/workspace/skills/spatial-agi-research/scripts/validate_thinking.sh
```

### 第3次失败

```bash
echo "❌ 验证失败3次，停止任务"
# 发送错误报告到群
# 退出任务
exit 1
```

## 常见偷懒模式及纠正

### 模式1: 缺少Mermaid图表

**检测**: `grep "```mermaid" | wc -l < 3`

**纠正**: 补充缺失的图表：
- 核心见解演进图（graph LR）
- 知识缺口分析（pie图）
- 10层架构思维导图（mindmap）

### 模式2: 缺少与昨天的联系

**检测**: `grep "与昨日思考的联系" 返回空`

**纠正**: 读取昨天的思考文档，生成联系章节

### 模式3: 文档过短

**检测**: `wc -l < 800`

**纠正**: 扩展每个章节的内容，确保深度

## 预计时间

- 1-2分钟（验证）
- 5-30分钟（如果需要补充）
